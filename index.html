MEET STAN

Stan is a mock AI chatbot for Stanbic Bank that provides customer service to the bankâ€™s clients through its conversational experience.

1. Prerequisites

For this project, I imported a couple of external libraries that include the following. All of them were installed using pip.

2. Setting up intents

The intents file is provided with tags where I specified the patterns of phrases a user may choose and their responses. I provide a couple of examples of how the user may interact with the chatbot to allow the bot to recognize more than one iterations of the same thing. Some examples include the following:

3. Import data

Here I initialized the variables to be used in the project (words, classes, documents, and letters to be ignored). I also imported the JSON file as text and passed that text to the loads function. This resulted in a JSON object that is used as a dictionary for this project.

The next step is tokenizing the patterns (breaking the whole text into tokens). After that, I iterated through the patterns and tokenized the sentences. I then append each word in the words list and create a list of classes for the tags.

After tokenization, I lemmatized the words to convert them into their lemma form(the stem) and eliminated the duplicates. I then created a pickle file to store the words and classes that will be used while predicting.

4. Prepare training data

In this step, I represented the words as numerical data that can be understood by the neural network. In order to achieve this I used a bag_of_words. This sets the individual word and classes to either 0 or 1 depending on whether they appear in the pattern.

The training data will provide our input and output. The input is the words we created in the JSON file, and the output is the class our input pattern belongs to.

I then shuffled the training data, and turned it into a numpy array. The last step is splitting the values into x and y values (features and labels) that will be used to train the neural model.

5. Build the neural model

I built a deep neural network that has three layers using the Keras sequential API. The activation function used for the project is the softmax function.

After training the model for 200 epochs on a learning rate of 0.01, I achieved 100% accuracy from the model. I then saved the model as chatbot_model.h5

6. Creating the chatbot

The model that was previously trained will only tell us the class that the response belongs to. In this phase, I implemented some functions to identify the class and then retrieve a random response from the list of responses.

7. The Graphical User Interface

I used HTML, CSS, and JAVASCRIPT to create a graphical user interface that is intuitive for the user. In addition to creating the GUI for the chatbot, I also added a mock website where my chatbot was embedded for a realistic feel. I then used Flask to connect the front end to the chatbot.

DIGITS

This project is a neural network that recognizes handwritten digits

1. Prerequisites

For this project, I imported a couple of external libraries that include the following. All of them were installed using pip.

2. Loading the data

For the network to be able to predict handwritten digits, it first needed to be trained on thousands of images. I used the MNIST data set for this purpose. The data set has a training set of 60,000 images and a test set of 10,000 images.

After determining the labels and features, I then normalized the x values. This step is necessary in order to make computation easier for the model.

3. Building the neural network

To build the neural network, I used tensorflow then defined the model and the individual layers. I created a Sequential model, which is a type of neural network that is defined layer by layer. My network had four layers. I started out with a Flatten layer for the input and two Dense layers which are our hidden layers. The last layer is the output layer, which is also another Dense layer.

4. Training the model

After building the model, I compiled it to define certain parameters and configure them for training and testing. While compiling I defined the optimizer, the loss function, and the accuracy metrics.

To train the model, I passed three parameters, the x-value, the y-value, and the number of epochs. The epochs define how many times the same data is fed to the model. I used 5 epochs in order to increase the accuracy. After the model was trained, I tested it for the loss and the accuracy.

5. Predicting handwritten digits

I used MS Paint to draw 28x28 pixel digits which I then saved to a directory named digits. The images were then loaded, predicted and visualized as shown.
